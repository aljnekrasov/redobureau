# ТЗ: тексты, блог (Journal), раздел Shop

Статус: черновик на согласование. Три независимых блока работ — можно
запускать по отдельности, приоритет предлагается А → Б → В.

Контекст: два сайта на одном коде (redobureau.com — en/es, redobureau.ru — ru),
контент file-based в `content/`, редактирование через панель на .com, синк
на .ru кнопкой. Всё новое обязано учитывать: мультиязычность (en/ru/es),
разделение аудиторий (поле `audience`), sitemap/hreflang (плагин seo),
кэш страниц.

## Блок А. Тексты на существующие страницы (SEO/AEO)

Цель: сайт — портфолио почти без текста; поисковикам и генеративным движкам
(ChatGPT/Perplexity) не за что зацепиться. Добавляем текстовые зоны, не
ломая минималистичный дизайн.

### А1. /studio — главная текстовая страница сайта

Сейчас: одно поле `text` (абзац) + 4 направления (`directions`) + команда.

Добавить в блюпринт `site/blueprints/pages/studio.yml` и шаблон:

| Поле | Тип | Объём | Зачем |
|---|---|---|---|
| `intro_extended` | textarea (kirbytext) | 800–1500 знаков | Развёрнуто: кто мы, для кого работаем, гео (Barcelona/BA/SP/Miami), чем отличаемся. Ключевые запросы: branding agency, packaging design studio + города |
| `clients` | structure (name, url) | 10–20 записей | Список клиентов — сущности для AEO («работали с Mail.ru, Faberlic…») |
| `awards` | structure (year, title, url) | сколько есть | Премии/публикации — доверие + сущности |
| `faq` | structure (question, answer) | 5–8 пар | Рендерится + JSON-LD `FAQPage` — прямой корм для ответных движков |

Рендер: после существующих направлений, той же типографикой (col-6/col-5,
классы уже есть). FAQ — простым списком без аккордеона (меньше JS).

Тексты пишет: заказчик (черновики может сгенерить Claude, вычитка
обязательна). Языки: en — мастер, es и ru — переводы. На .ru формулировки
адаптируются («Студия Редо», города — по позиционированию рынка).

### А2. Главная (3D-страница)

Ниже вьюпорта с 3D добавить один SEO-блок: поле `seo_text` (500–800 знаков,
kirbytext) — «студия брендинга и дизайна: услуги, рынки, ссылки на /work и
/studio». Внутренние ссылки обязательны. Визуально — мелкий текст в футерной
зоне, не конкурирует с анимацией.

### А3. Страницы проектов

Поля уже есть, работа контентная:
- заполнить пустые `Sharetext` (og-описания) — ~5 проектов без них;
- в конце `Desc` каждого проекта добавить 1 предложение-результат
  («сайт поднял конверсию…», «тираж распродан…») — где есть факты;
- у всех проектов проверить `audience` (ru-only компании скрыть с .com).

### А4. Contacts

Добавить поле `offices` (structure: city, country, address, timezone) —
рендер списком + JSON-LD `Organization.location`. Сейчас адрес один (CABA).

Оценка Блока А: код+шаблоны ~0.5 дня; тексты en+перевод es/ru — основная
работа, на стороне заказчика.

## Блок Б. Блог → раздел «Journal»

### Что уже есть в коде (недоделанный задел)
- блюпринты `pages/stories.yml` (листинг) и `pages/story.yml` (статья,
  builder: текст+галереи, related-проекты, cover/previewCover);
- сниппеты `blocks/block-text.php`, `blocks/block-pictures.php`;
- шаблон `stories.php` — ПУСТОЙ файл, шаблона `story.php` нет, контента нет.

### Целевое устройство

**URL**: `/en/journal`, `/es/journal`, `/ru/journal` (папка
`content/journal/`, статьи — подстраницы `journal/2026-07-slug/`).
Имя «journal» вместо «stories» — решить (см. Открытые вопросы).

**Блюпринт статьи** — доработать `story.yml`:
- добавить: `date` (обяз.), `intro` (textarea, 200–300 знаков — анонс и
  meta description), `audience` (как у проектов: intl/ru — русскоязычные
  кейсы-тексты не светим на .com), `main_title`/`og_title`/`sharetext`
  (переиспользовать `tabs/meta.yml`);
- builder дополнить блоком `block-quote` (цитата) и `block-video` (Vimeo id);
- `Autors` → исправить на `Authors`.

**Шаблоны** (новые):
- `stories.php` (листинг): карточки — previewCover, дата, заголовок, intro;
  фильтр по `audienceAllows()`; пагинация по 12; сортировка по `date` desc;
- `story.php` (статья): h1, дата, авторы, cover, builder-блоки, related
  (уже в блюпринте), «следующая статья» по аналогии с проектами.

**SEO/инфраструктура** (правки существующих файлов):
- sitemap: статьи попадут автоматически (listed + audience уже
  учитываются) — проверить;
- JSON-LD `Article` в story.php (headline, datePublished, author, image);
- hreflang — работает из коробки (header общий);
- RSS `/journal/feed` — роут в seo-плагине (для рассылок/агрегаторов), v2;
- rbTrack событие `story_view` — в аналитический слой (аудитория
  «читатели» для ретаргета).

**Навигация**: ссылка «Journal» в header ставится за конфиг-флагом
`site.nav.journal => true|false` per host — включаем, когда будет ≥3
статьи. До включения раздел доступен по прямому URL (для проверки),
статьи в статусе draft не видны вообще.

**Редакционный процесс**: статья пишется на en в панели → переводы
es/ru дополняются файлами `.es.txt`/`.ru.txt` (или в панели). Для
ru-only материалов — audience=ru, пишутся сразу на русском.

Оценка Блока Б: шаблоны+блюпринт+SEO ~1–1.5 дня кода. Дальше — контент.

## Блок В. Shop (скрытый до запуска)

### Этап V1 — каталог без оплаты (это ТЗ)

**Структура**: `content/shop/` (страница-витрина) + `shop/<slug>/`
(товар). URL `/en/shop`, `/ru/shop`.

**Блюпринт товара** `pages/product.yml`:

| Поле | Тип | Примечание |
|---|---|---|
| `title` | text | название |
| `price_intl` | number + `currency_intl` (select: USD/EUR) | цена для .com |
| `price_ru` | number (RUB) | цена для .ru — **цены и валюты разводятся по доменам**, как audience |
| `status_stock` | select: available / preorder / soldout | бейдж на карточке |
| `sizes` | tags | опционально (мерч) |
| `desc` | textarea | описание |
| `gallery` | files | фото товара |
| `audience` | checkboxes intl/ru | какие товары на каком сайте (переиспользуем плагин) |

**Шаблоны**: `shop.php` (грид карточек, повторяет пластику /work) и
`product.php` (галерея, цена своего домена, описание, кнопка).

**Кнопка покупки в V1 — без эквайринга**: «Order» открывает mailto или
форму (имя, контакт, товар, размер) → письмо на hello@. Продажи руками,
пока не понятен объём. Это позволяет запустить витрину за дни, а не недели.

**Скрытие до запуска** (требование «сделать, но скрыть»):
- в nav не добавляем вовсе (флаг `site.nav.shop => false` per host);
- страницы в статусе `unlisted` → не попадают в sitemap (там только
  listed) и не находятся поиском по сайту;
- в `shop.php`/`product.php` — `<meta name="robots" content="noindex">`
  пока `site.nav.shop === false`;
- доступ для проверки — по прямому URL.

### Этап V2 — оплата (отдельное ТЗ после V1, здесь только развилки)

- **.com**: Stripe Payment Links (без бэкенда, ссылка на товар) →
  простейший старт; либо Snipcart/Shopify Buy Button, если нужна корзина.
- **.ru**: ЮKassa / Robokassa. ⚠️ 54-ФЗ: онлайн-касса и фискальные чеки
  обязательны (у ЮKassa есть решения «касса в аренду»). Плюс оферта,
  доставка (СДЭК/Почта), возвраты — юридический блок работ.
- Учёт остатков: в V1/V2 — поле в панели руками; складской учёт не строим.

Оценка V1: блюпринты+шаблоны+скрытие ~1 день кода + фото/описания товаров.

## Общие механики (одна реализация на все три блока)

1. **Флаги навигации** в конфигах хостов:
   `site.nav => ['journal' => false, 'shop' => false]` — header читает
   флаги; включение раздела = одна строка в конфиге + git pull.
2. **Noindex для скрытых разделов** — сниппет в header по тем же флагам.
3. Всё новое — через `t(...)` (никаких хардкодов языка), с `audience`,
   с учётом кэша страниц (никакой персонализации в PHP-рендере).

## Порядок работ (предложение)

1. А1+А2 код (поля+рендер) — быстро, разблокирует написание текстов
2. Тексты Блока А (заказчик ± Claude-черновики) → перевод → выкладка
3. Блог: код целиком → 3 стартовые статьи → включение в nav
4. Shop V1 код → фото/цены → внутренняя проверка → решение по V2
